/Goal Create a modern replacement for MediaShow v0.9.5 — a Total Commander lister plugin (.wlx/.wlx64) for viewing/playing multimedia files. The original plugin (Delphi 6, 2006) has visual bugs on modern Windows. The new plugin should support modern formats, have a proper UI with seekbar, volume control, playlist, and work in both F3 lister mode and QuickView mode.

Discoveries
1. TC lister window intercepts mouse messages — trackbar controls don't receive WM_HSCROLL when MFPlay creates its own rendering window inside hVideoWnd. Original MediaShow uses DirectShow IVideoWindow with put_Owner to render into existing panel, NOT MFPCreateMediaPlayer.
2. Trackbar rendering issue — TrackBar controls created in WM_CREATE don't render until mouse interaction. Creating them via PostMessage(WM_USER+500) after window is shown fixes rendering, but then toolbar creation via CreateControls() somehow breaks it. Current approach: create ALL controls together in CreateControls() called from WM_CREATE, no PostMessage split.
3. Prev/Next crash fixed — MFPlayer_Open was not stopping event thread before releasing old resources. Added WaitForSingleObject on event thread before cleanup.
4. Playlist from selected files — FindWindowEx(hTC, NULL, "SysListView32", NULL) returns NULL. TC's lister API doesn't provide selected files directly. Current implementation scans directory for media files (FindFirstFile/FindNextFile).
5. TC API key facts:
   - TC subclasses plugin window for 'n'/'p' hotkeys
   - ListNotificationReceived forwards WM_COMMAND from toolbar buttons
   - itm_next tells TC to switch to next file
   - Tab/arrows/F3 are consumed by TC — cannot intercept
   - TC uses Lazarus/Free Pascal (NOT Delphi) — file panel is LCLListBox, not SysListView32
   - LB_GETTEXT returns "filename size DD.MM.YYYY HH:MM attributes" format
   - cm_CopyFullNamesToClip does NOT work during ListLoad (TC processes synchronously)
6. Original MediaShow architecture:
   - Lister mode (F3): child window inside TC lister panel
   - Independent player: separate popup window
   - Playlist embedded in center area during audio playback (not popup)
   - DirectShow IVideoWindow with put_Owner for video rendering
   - Uses Delphi VCL (TForm, TPanel, TTrackBar, TListView)
7. Volume slider direction: TBS_VERT range 0-100 means up=100 (max). User wants horizontal slider.
8. Mouse wheel: Should ONLY control volume near the volume slider, NOT seek.
